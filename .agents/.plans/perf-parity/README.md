## Source findings (vetted by the advisor against the code at `b5da256`)

- **PERF-01/PERF-02 → plan 001**: `getBlockSums()` has zero production callers (grep-
  verified); `calculateVisibleRange` walks from index 0 (`virtualList.ts:117-124`); all
  four `getScrollOffsetForIndex` production call sites omit `blockSums`. Also folded in:
  PERF-03 (`updateDebugTailDistance` querySelector + forced layout every scroll frame,
  `SvelteVirtualList.svelte:636`) because it pollutes the plan's own measurements.
- **PARITY-02 → plan 002**: range/atTop/atBottom already computed but only deliverable
  via debug mode (`types.ts:94-104`, `SvelteVirtualList.svelte:725-753`); all five
  comparison libraries expose this publicly.
- **PARITY-03 → plan 003**: align union lacks `center` (`types.ts:109`, requested in
  issue #165); no offset-based scroll (issue #66); ecosystem-standard in TanStack /
  tiny-virtual-list / virtuoso.

## Findings considered and NOT planned (so nobody re-audits them)

- **PARITY-01 stable item keys** (`#each` keyed by `originalIndex`,
  `SvelteVirtualList.svelte:1021`; height cache keyed by index): the single biggest
  parity gap (prepend/insert/reorder corrupts the height cache and remounts everything
  below the mutation), but M–L effort with MED risk to the anchor hot path — it deserves
  its own initiative with a design pass, not a slot in a top-3 batch. Strongly
  recommended as the NEXT batch; it is also the prerequisite for prepend-safe
  bidirectional loading (PARITY-04).
- **PARITY-04 start-edge loading / prepend anchoring**: blocked on stable keys
  (PARITY-01); also strategically ambiguous — reverse/chat was deliberately removed
  (#395) and delegated to `@humanspeak/svelte-virtual-chat`, so decide where prepend
  support belongs before building.
- **PARITY-05 horizontal orientation**: real gap (the docs site's own comparison matrix
  concedes it; issue #308), but L effort touching every hot path; demand signal is one
  closed issue. Revisit as a design spike if demand grows.
- **PARITY-06 window/document scrolling**: strong ecosystem expectation, no direct issue
  requesting it; MED confidence. Investigate demand before building.
- **PERF-04 write-only `_measuredFlags` Uint8Array** (reallocated per items-length
  change, `ReactiveListManager.svelte.ts:465,542,589`; grep-verified never read): real
  but small; fold into any future manager PR rather than a standalone plan.
- **PERF-05 `displayItems` wrapper-object churn** (`SvelteVirtualList.svelte:592-601`):
  O(window) allocation per range shift; low impact next to 001. Candidate quick win.
- **PERF-06 sparse `Record` height cache**: constant-factor; deliberately deferred until
  after 001 lands — the remaining walks are ≤ blockSize, which may make this moot.
  Re-measure before doing it.
- **Grid/masonry**: covered by an existing maintainer decision (issue #396: separate
  package pending community interest) — not a finding per the audit playbook's
  settled-decision rule.
